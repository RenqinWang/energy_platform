# 阶段6工作总结 - 前端数据展示实现

## 概述

本阶段实现了基于HTML/CSS/JavaScript的Web前端界面，用于可视化展示能源平台数据。前端通过RESTful API与后端通信，提供交互式图表、数据表格和多维度过滤功能。

**实施时间**: 2024年（阶段6）  
**技术栈**: HTML5, CSS3, JavaScript ES6, Chart.js 4.4.0, Axios 1.6.0  
**架构**: 纯前端应用，无需构建工具

---

## 一、功能特性

### 1.1 核心功能

1. **实时数据可视化**
   - 日报表能耗与供冷量趋势图
   - 小时供能曲线（功率与温度）
   - COP趋势分析图
   - 经济指标分析图（成本、收入、利润）

2. **多维度数据过滤**
   - 按站点过滤
   - 按设备过滤
   - 按日期范围过滤
   - 实时查询和刷新

3. **状态卡片**
   - 总能耗统计
   - 总供冷量统计
   - 平均COP
   - 净利润汇总

4. **数据表格**
   - 日报表详细数据
   - 小时数据详细数据
   - 表格视图切换
   - CSV导出功能

5. **响应式设计**
   - 支持桌面和移动设备
   - 自适应布局
   - 触摸友好的交互

## 四、图表详解

### 4.1 日报表趋势图

**类型**: 折线图（双Y轴）  
**数据源**: gold_report_daily  
**X轴**: 日期  
**Y轴**: 
- 左Y轴: 能耗 (kWh) - 红色
- 右Y轴: 供冷量 (kWh) - 蓝色

**用途**: 展示能耗和供冷量随时间的变化趋势，便于识别异常波动。

### 4.2 小时供能曲线

**类型**: 折线图（双Y轴）  
**数据源**: gold_supply_curve_hourly  
**X轴**: 时间（月-日-小时）  
**Y轴**:
- 左Y轴: 平均功率 (kW) - 橙色
- 右Y轴: 供水温度 (°C) - 绿色

**数据范围**: 最近48小时  
**用途**: 展示设备功率和温度的小时级变化，用于精细化运行分析。

### 4.3 COP趋势图

**类型**: 柱状图 + 折线图（双Y轴）  
**数据源**: gold_report_daily  
**X轴**: 日期  
**Y轴**:
- 左Y轴: COP - 紫色柱状图
- 右Y轴: 运行率 (%) - 粉色折线

**用途**: 分析设备能效（COP）与运行率的关系，优化运行策略。

### 4.4 经济指标图

**类型**: 柱状图 + 折线图  
**数据源**: gold_report_daily  
**X轴**: 日期  
**Y轴**: 金额 (元)
- 能源成本 - 红色柱状图
- 供冷收入 - 绿色柱状图
- 净利润 - 蓝色折线

**用途**: 展示经济效益，评估运营成本和收益。

## 五、交互功能

### 5.1 过滤器联动

- 选择站点后，设备下拉框自动更新为该站点的设备
- 清空站点选择，设备下拉框恢复显示所有设备
- 日期范围默认为最近30天，可自定义

### 5.2 数据查询

- 点击"查询数据"按钮，应用当前过滤条件重新加载数据
- 点击"刷新"按钮，使用当前过滤条件刷新数据
- 查询时显示加载动画，完成后自动隐藏

### 5.3 表格切换

- 点击"日报表"标签，显示日报表数据
- 点击"小时数据"标签，显示小时数据（最近100条）
- 当前激活的标签高亮显示

### 5.4 CSV导出

- 点击"导出CSV"按钮，下载当前表格视图的数据
- 文件名自动包含日期：`daily_report_YYYY-MM-DD.csv` 或 `supply_curve_YYYY-MM-DD.csv`
- CSV文件包含UTF-8 BOM，确保Excel正确显示中文

### 5.5 图表交互

- 鼠标悬停显示详细数值
- 点击图例可隐藏/显示对应数据系列
- 图表自动响应窗口大小变化

## 十、故障排查

### 10.1 "无法连接到服务器"错误

**原因**: 后端API未运行或地址不正确

**解决方案**:
1. 检查API服务: `curl http://localhost:8000/health`
2. 启动API服务: `cd /home/student/energy-platform/backend && ./start_api.sh`
3. 检查 `js/config.js` 中的API地址配置

### 10.2 CORS错误

**原因**: 浏览器阻止跨域请求

**解决方案**:
1. 使用HTTP服务器运行前端（不要直接打开HTML文件）
2. 确保后端CORS配置包含前端地址（已在 `backend/config.py` 中配置）

### 10.3 图表不显示

**原因**: Chart.js库未加载或数据格式错误

**解决方案**:
1. 检查浏览器控制台是否有JavaScript错误
2. 确保网络连接正常（Chart.js从CDN加载）
3. 检查API返回的数据格式是否正确

### 10.4 数据表格为空

**原因**: 查询条件过滤掉了所有数据

**解决方案**:
1. 调整日期范围，包含有数据的日期
2. 清空站点和设备过滤条件，查看所有数据
3. 运行 `backend/test_api.py` 验证数据是否存在
