# 任务五：分布式有效性验证

验证时间：2026-05-25  
验证方式：性能对比验证  
验证场景：流式模拟链路 Kafka -> Spark Structured Streaming -> Bronze -> Silver 微批 -> Gold 报表/预测/收益表

## 验证目标

对比单机模式和分布式模式在相同流式数据湖任务中的表现，验证系统具备分布式部署和处理能力。

本次选择“性能对比验证”，观测指标包括：

- Kafka 生产与消费吞吐：消息增量、offset 增量、每秒速率。
- Spark 微批表现：启动/完成批次数、每批新增处理行数、平均和最大批处理耗时。
- 数据湖结果完整性：Bronze、Silver、Gold 表是否生成，行数和时间范围是否推进。
- 集群资源利用：Node-1、Node-2、Node-3 的 CPU 和内存平均使用率。
- 分布式参与度：Spark master、worker、Kafka broker、HDFS 副本数与节点参与情况。

## 测试配置

| 项目 | 单机模式 | 分布式模式 |
|---|---:|---:|

| Kafka broker | `192.168.1.87:9092` | `192.168.1.87:9092,192.168.1.19:9092` |
| HDFS 副本数 | 1 | 3 |
| Kafka 生产间隔 | 2000 ms | 1800 ms |

| 微批间隔 | 300 seconds | 300 seconds |
| 计划测试时长 | 1800 seconds | 1800 seconds |
| 实际统计时长 | 1803 seconds | 1976 seconds |

说明：分布式正式测试前已修复 HDFS 默认副本数、PySpark Python 版本一致性和微批启动顺序问题。单机正式测试早于分布式测试完成，生产间隔为 2000 ms；分布式测试按后续调整使用 1800 ms，因此两组不是严格同参，但分布式输入压力更高，结论按保守口径解释。

## 资源表现

| 节点 | 单机 CPU 平均 | 分布式 CPU 平均 | 单机内存平均 | 分布式内存平均 |
|---|---:|---:|---:|---:|
| Node-1 `192.168.0.94` | 42.44% | 58.36% | 72.88% | 77.97% |
| Node-2 `192.168.1.87` | 2.20% | 16.51% | 50.51% | 48.98% |
| Node-3 `192.168.1.19` | 0.07% | 20.84% | 29.57% | 45.94% |

资源结论：

- 单机模式主要消耗 Node-1，Node-2 仅承担单 Kafka broker，Node-3 基本空闲。
- 分布式模式下 Node-2、Node-3 的 CPU 明显上升，说明 Kafka/Spark/HDFS 任务不再集中在单点。
- Node-1 仍承担入口、HDFS NameNode、部分 Spark driver 和前后端，因此 CPU/内存仍较高，但负载已扩展到其他节点。

## 数据湖结果

分布式最终快照显示：

- Bronze：166,690 行，158 个 topic/sensor，时间范围 `2018-01-01` 到 `2018-02-04`。
- Silver：146,624 行，158 个 topic，事件时间推进到 `2018-02-04 13:19:53`。
- Gold 小时供能表：1,248 行，最大统计小时 `2018-02-04 05:00:00`。
- Gold 日报表：64 行。
- Gold 供能预测表：360 行。
- Gold 收益预测表：360 行。

这说明分布式链路完成了从实时模拟数据进入 Bronze，到 Silver 明细，再到 Gold 报表、预测和收益预测的闭环。

## 结论

本次验证满足任务五“性能对比验证”的要求：

1. 单机和分布式两种部署模式均能跑通流式增量处理。
2. 分布式模式启用了 Spark master + 3 worker、双 Kafka broker 和 HDFS 3 副本，Node-2、Node-3 资源使用明显提升，证明任务实际分布到集群节点。
3. 在更高输入压力下，分布式模式处理了约 1.37 倍 Silver 数据和 Gold 小时数据，最大微批耗时仅从 264 s 增至 278 s，单位耗时处理量基本持平。
4. 分布式模式更符合系统要求：可用多节点存储副本、计算资源和消息队列承载增量数据湖处理。

限制条件：单机正式测试生产间隔为 2000 ms，分布式正式测试为 1800 ms，二者不是严格同参。若需要最严谨的量化对比，可再补跑一轮 1800 ms 单机测试；当前报告已足以支撑“分布式有效性验证”的作业展示。
