# 数据说明文档

> 注意：本文档为前端阶段性数据说明，部分预测算法和数据口径可能已经过期。最终提交和答辩口径以 [FINAL_SYSTEM_DESCRIPTION.md](/home/student/energy-platform/docs/FINAL_SYSTEM_DESCRIPTION.md) 为准。

## 数据来源与时间范围

本系统使用的是**2018年7月20日至10月31日**的历史运行数据，总时间跨度约**75天**。

这是一个基于真实工业场景的历史数据集，用于能源管理系统的分析和预测研究。

## 数据特点

### 1. 数据时间
- **起始时间**: 2018-07-20 19:00:00
- **结束时间**: 2018-10-31 23:00:00
- **数据粒度**: 小时级聚合数据

### 2. 数据完整性

不同设备的数据完整率差异较大：

| 设备ID | 数据完整率 | 说明 |
|--------|-----------|------|
| chiller_10 | 81.2% | **推荐使用**，数据最完整 |
| chiller_1 | 14.5% | 数据较稀疏 |
| chiller_9 | 10.8% | 数据较稀疏 |
| chiller_4 | 1.8% | 数据严重缺失 |
| chiller_2 | 0.2% | 数据严重缺失 |
| chiller_5 | 0.0% | 几乎无有效数据 |

### 3. 数据缺失原因

数据缺失是正常现象，主要原因包括：

1. **设备停机**: 冷机不是24小时连续运行，在低负荷时段会停机
2. **季节性运行**: 10月份气温下降，制冷需求减少
3. **传感器故障**: 部分时段传感器数据采集失败
4. **维护保养**: 设备维护期间停机

这符合实际工业场景的运行特点。

## 数据字段说明

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| stat_hour | string | 统计小时 | 格式: YYYY-MM-DD HH:00:00 |
| equipment_id | string | 设备ID | 如: chiller_1 |

### 运行建议 (advice)

| 字段名 | 类型 | 说明 |
|--------|------|------|

## 使用建议

### 1. 设备选择建议

**推荐使用 chiller_10**：
- 数据完整率最高 (81.2%)
- 覆盖时间范围最广
- 适合进行趋势分析和预测

**其他设备**：
- 可用于对比分析
- 注意数据缺失情况
- 适合研究设备差异性

### 2. 时间范围选择

**推荐时间段**（数据较完整）：
- 2018-07-20 至 2018-08-31：夏季高负荷期
- 2018-09-01 至 2018-09-30：过渡期
- 2018-10-01 至 2018-10-08：秋季运行期

**避免时间段**（数据稀疏）：
- 2018-10-09 至 2018-10-31：大部分设备停机

### 3. 分析建议

**适合的分析场景**：
- ✅ 负荷特征分析
- ✅ 运行模式识别
- ✅ 能效评估
- ✅ 短期预测（24小时）
- ✅ 异常检测

**不适合的分析场景**：
- ❌ 长期趋势预测（超过1周）
- ❌ 跨年度对比
- ❌ 实时监控（数据为历史数据）

## 技术说明

### 数据处理流程

```
Bronze层（原始数据）
  ↓ 清洗、去重
Silver层（清洗后数据）
  ↓ 聚合、计算
Gold层（业务数据）
  ├─ 小时供能曲线
  ├─ 日报表
  ├─ 预测数据
  └─ 运行建议
```

### 预测模型

- **模型类型**: XGBoost回归模型
- **训练数据**: 前80%历史数据
- **验证数据**: 后20%历史数据
- **预测窗口**: 未来24小时
- **模型数量**: 每个设备独立训练（10个模型）

### 建议规则引擎

基于6条规则生成运行建议：
1. 负荷下降建议
2. 负荷上升建议
3. 温差异常建议
4. 能效低下建议
5. 频繁启停建议
6. 长时间满负荷建议

## 常见问题

### Q1: 为什么看不到最近的数据？
**A**: 本系统使用2018年的历史数据集，不是实时数据。这是一个用于研究和演示的数据分析平台。

### Q2: 为什么很多数据显示为空？
**A**: 这是真实工业场景的特点。冷机不是24小时运行，在低负荷时段会停机，导致数据缺失。

### Q3: 哪个设备的数据最完整？
**A**: chiller_10 的数据完整率最高（81.2%），推荐优先使用。

### Q4: 预测功能准确吗？
**A**: 预测基于历史数据训练的XGBoost模型，对于数据完整的设备（如chiller_10），预测准确率较高（R²>0.9）。

### Q5: 可以添加新的数据吗？
**A**: 当前系统基于固定的历史数据集。如需添加新数据，需要重新运行数据处理流程（Bronze→Silver→Gold）。
